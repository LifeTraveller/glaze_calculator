# 数据存储说明

## 数据持久化

本应用将数据存储在设备本地。**注意：清除应用数据或卸载应用后数据会被删除。**

## 存储位置

### Web 平台（浏览器/GitHub Pages）

- 位置: 浏览器 **IndexedDB**（SQLite via sql.js/WASM）
- 数据库: `glaze_calculator.db`（存储于 IndexedDB 中）
- 图片: 以 **base64** 编码直接存储在数据库内，无独立图片目录
- 说明:
  - 数据存储在**当前设备的当前浏览器**中，换设备或换浏览器数据不互通
  - 清除浏览器缓存/网站数据会删除所有配方
  - iOS Safari 若长时间不访问，系统可能自动清理（建议定期导出备份）

### Android 平台
- 位置: 应用内部存储目录 (应用专属目录)
- 数据库: `glaze_calculator.db`
- 图片目录: `recipe_images/`
- 说明: 卸载应用后数据**会被**删除

### Windows 平台
- 位置: `C:\Users\<用户名>\AppData\Roaming\<应用包名>\`
- 数据库: `glaze_calculator.db`
- 图片目录: `recipe_images\`

### macOS 平台
- 位置: `~/Library/Application Support/<应用包名>/`
- 数据库: `glaze_calculator.db`
- 图片目录: `recipe_images\`

### Linux 平台
- 位置: `~/.local/share/<应用包名>/`
- 数据库: `glaze_calculator.db`
- 图片目录: `recipe_images\`

## 数据备份

**重要：数据仅保存在本地，建议定期导出备份！**

应用提供了数据导出/导入功能：

### 导出数据
1. 打开"配方列表"界面
2. 点击右上角的下载图标
3. 配方数据会以 JSON 格式导出
   - **Web 端**：浏览器自动下载 JSON 文件到本地
   - **原生端**：通过系统分享功能选择保存位置

### 导入数据
1. 打开"配方列表"界面
2. 点击右上角的上传图标
3. 选择之前导出的 JSON 文件
4. 配方数据会自动导入到应用中

## 注意事项

- 导出的 JSON 文件**不包含图片**，仅包含配方的矿物配比数据
- **Web 端**：强烈建议定期导出备份，浏览器数据随时可能被清除
- **原生端**：卸载应用会删除所有数据，重装后可通过导入功能恢复
- 建议将导出的备份文件保存到云盘或电脑上
